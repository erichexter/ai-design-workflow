# Feature Definition Workflow

Use this workflow to convert a rough feature idea into a design ready for implementation.

---

Step 1 — Define the problem  
Prompt: [prompts/discovery/define-problem.md](../prompts/discovery/define-problem.md)

Step 2 — Generate product requirements  
Prompt: [prompts/requirements/generate-product-requirements.md](../prompts/requirements/generate-product-requirements.md)

Step 3 — Generate user stories  
Prompt: [prompts/requirements/generate-user-stories.md](../prompts/requirements/generate-user-stories.md)

Step 4 — Generate acceptance criteria  
Prompt: [prompts/requirements/generate-acceptance-criteria.md](../prompts/requirements/generate-acceptance-criteria.md)

Step 5 — Identify edge cases  
Prompt: [prompts/review/identify-edge-cases.md](../prompts/review/identify-edge-cases.md)

Step 6 — Propose architecture options  
Prompt: [prompts/architecture/propose-architecture-options.md](../prompts/architecture/propose-architecture-options.md)

Step 7 — Generate API spec  
Prompt: [prompts/architecture/generate-api-spec.md](../prompts/architecture/generate-api-spec.md)

Step 8 — Generate implementation plan  
Prompt: [prompts/planning/generate-implementation-plan.md](../prompts/planning/generate-implementation-plan.md)

---

## Notes

- Each step takes the output of the previous step as input.
- You can start mid-workflow if earlier artifacts already exist.
- Run Step 5 (edge cases) before committing to an architecture in Step 6.
